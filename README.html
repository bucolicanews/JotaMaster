Modelagem de Dados & RLS

O sistema utiliza o Supabase (PostgreSQL) com uma arquitetura Multi-tenant baseada em Row Level Security. Abaixo, a estrutura das tabelas principais:

profiles

Extensão do Auth.Users. Armazena roles (admin, empresa) e status da conta.

  • id (UUID) - PK
  • role (TEXT) - RBAC
  • status (TEXT) - active/suspended

system_modules

Catálogo global do Marketplace. Define se o módulo é nativo ou externo (CDN).

  • id (TEXT) - PK
  • module_type (internal/iframe)
  • bundle_url (TEXT)

Segurança Zero Trust (RLS)

-- Exemplo de Política de Isolamento de Tenant
CREATE POLICY "Usuários veem apenas seus dados" 
ON public.ai_skills 
FOR SELECT 
USING (auth.uid() = user_id OR is_global = true);
